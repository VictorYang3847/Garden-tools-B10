# 可靠性工具改版 - Product Requirement Document

## Overview
- **Summary**: 将现有单功能 B10 寿命分析工具升级为综合性可靠性工程工具平台，包含 FMEA、寿命数据分析、可靠性预测、测试计划评估、故障树分析、可靠性增长、维护可用性、降额裕度分析、环境适应性分析、数据管理与报告共 10 大核心模块。
- **Purpose**: 为可靠性工程师提供一站式工具集，覆盖产品全生命周期的可靠性分析需求，从设计阶段的 FMEA 到量产阶段的可靠性增长验证。
- **Target Users**: 可靠性工程师、品质工程师、设计工程师、测试工程师、项目经理

## Goals
- 提供 10 大可靠性分析模块，覆盖 FMEA、寿命分析、可靠性预测、测试评估、FTA、可靠性增长、维护分析、降额分析、环境分析、数据管理
- 保持现有 B10/Weibull 分析功能，作为寿命数据分析模块的子功能
- 统一的项目/产品/型号层级数据组织
- 现代化 UI 设计，支持深色主题
- 纯前端实现，数据本地存储，支持导入导出
- 模块化架构，便于后续扩展

## Non-Goals (Out of Scope)
- 后端服务与云端数据同步
- 多人协作与权限管理
- 高级仿真计算（有限元、CFD 等）
- 移动端 APP（仅响应式 Web）
- 与 PLM/ERP 系统集成
- 付费订阅与商业化
- AI/ML 智能推荐

## Background & Context
- 现有项目为园林电动工具 B10 分析工具，仅支持 Weibull 分布的 B10 寿命计算
- 三阶段工作流（产品定义→测试规划→结果分析）已完成
- 技术栈：原生 HTML/CSS/JS + ES Modules，无框架依赖
- 数据存储：localStorage + JSON 导入导出
- 部署方式：GitHub Pages 静态托管
- 目标：v3.0 全面升级为可靠性工具平台

## Functional Requirements

### FR-1: 统一导航与布局框架
- 左侧导航栏，展示 10 大模块
- 顶部项目/产品/型号选择器
- 面包屑导航
- 响应式布局，支持移动端

### FR-2: 失效模式与影响分析 (FMEA)
- 支持 DFMEA（设计）和 PFMEA（过程）两种类型
- 失效模式、失效后果、潜在原因、现行控制措施录入
- 严重度 (S)、发生度 (O)、探测度 (D) 1-10 分评分
- RPN = S × O × D 自动计算
- Action Priority 矩阵（AIAG-VDA 标准：高/中/低）
- 建议措施与责任分配
- 措施实施跟踪与重新评估

### FR-3: 可靠性预测与计算
- MTBF（平均无故障时间）预测
- 基于 MIL-HDBK-217、Telcordia SR-343、IEC 62309 标准的失效率计算
- 部件应力分析（温度、电流、电压降额因子）
- 系统级可靠度函数 R(t) 计算
- 串联/并联/混联系统模型
- 常用元器件失效率数据库（电阻、电容、半导体、连接器等）

### FR-4: 寿命数据分析
- 威布尔分析（Weibull Distribution）— 继承现有 B10 功能
- 指数分布、对数正态分布拟合
- 参数估计方法：MLE（最大似然估计）、RRX（X 方向秩回归）、RRY（Y 方向秩回归）
- 概率图（PP 图）可视化
- B10/B50 寿命计算
- 置信区间估计
- 失效模式/零件分解分析

### FR-5: 测试计划与评估
- ALT（加速寿命试验）计划管理
- HALT（高加速寿命试验）记录
- 样本量计算（基于置信度和可接受失效数）
- 试验方案设计（定时截尾/定数截尾/完全失效）
- 加速因子计算（Arrhenius、Coffin-Manson 模型）
- 试验数据收集与报告生成
- 台架条件记录

### FR-6: 故障树分析 (FTA)
- 顶事件定义
- 故障树构建：AND 门、OR 门、表决门、基本事件、中间事件
- 最小割集求解
- 定性分析（结构重要度）
- 定量分析（顶事件概率、概率重要度、关键重要度）
- 图形化故障树展示（树形结构）
- 事件库与复用

### FR-7: 可靠性增长与验证
- Duane 模型
- Crow-AMSAA 模型
- 可靠性增长曲线绘制
- RGV（可靠性增长验证）试验设计
- 达标的统计检验（MIL-STD-781、GJB 899）
- 失效数据时间序列分析
- 增长目标跟踪

### FR-8: 维护与可用性分析
- 预防性维护策略优化
- 可用度计算：固有可用度 A、可达可用度 Av、使用可用度 AP
- 备件需求预测（基于失效率和保障策略）
- 维修时间分布建模（对数正态分布）
- MTTR（平均修复时间）、MTBF、MTBM 计算
- 可用性框图

### FR-9: 降额与裕度分析
- 元器件降额检查（电阻、电容、电感、半导体、连接器等）
- 热裕度分析（最高工作温度 vs 额定温度）
- 电气裕度分析（电压、电流、功率降额比）
- 降额标准库：Mil-Hdbk-217、GJB/Z 35、企业自定义
- 降额等级：Ⅰ级（最大降额）、Ⅱ级（中等降额）、Ⅲ级（最小降额）
- 降额合规报告

### FR-10: 环境适应性分析
- 温度循环分析（高低温循环次数 vs 寿命损伤）
- 热冲击分析
- 振动与机械载荷分析（正弦振动、随机振动）
- 湿度、盐雾、粉尘等环境应力映射
- 环境应力筛选 (ESS) 方案
- 环境试验标准库（IEC 60068、GJB 150、MIL-STD-810）
- 环境因子计算

### FR-11: 数据管理与报告
- 项目/产品/型号三级组织架构
- 数据导入导出（Excel、CSV、JSON）
- 标准可靠性报告模板自动生成（Word/PDF 格式）
- 版本管理与变更追踪
- 模板库（FMEA 模板、测试计划模板、报告模板）
- 跨模块数据联动（FMEA 项 → 测试项 → 寿命分析）

## Non-Functional Requirements

- **NFR-1**: 纯前端实现，无需后端服务器
- **NFR-2**: 支持现代浏览器（Chrome、Edge、Firefox、Safari 最新两个版本）
- **NFR-3**: 页面首屏加载 < 2s（本地存储场景）
- **NFR-4**: 计算响应 < 500ms（1000 条数据以内）
- **NFR-5**: 深色主题 + 高对比度，符合专业工具视觉规范
- **NFR-6**: 代码模块化，单文件不超过 500 行
- **NFR-7**: 数据持久化在 localStorage，支持导出备份

## Constraints
- **技术**: 原生 HTML/CSS/JavaScript + ES Modules，不引入前端框架
- **部署**: GitHub Pages 静态托管
- **存储**: 浏览器 localStorage（单域名 5MB 限制）
- **性能**: 纯客户端计算，大数据量需考虑 Web Worker
- **时间**: 第一阶段完成核心框架 + 4 个优先模块

## Assumptions
- 用户具备基本的可靠性工程知识
- 单用户使用场景，无需多用户协作
- 数据量在 localStorage 容量范围内（< 5MB）
- 计算量在浏览器可接受范围内（非超大规模仿真）
- 用户使用桌面端为主，移动端为辅助查看

## Acceptance Criteria

### AC-1: 统一导航框架
- **Given**: 用户打开应用
- **When**: 页面加载完成
- **Then**: 显示左侧导航栏（10 个模块）、顶部项目选择器、主内容区
- **Verification**: `human-judgment`

### AC-2: 模块间切换
- **Given**: 用户在任意模块页面
- **When**: 点击左侧导航的另一个模块
- **Then**: 主内容区切换到对应模块，URL hash 更新，当前选中状态高亮
- **Verification**: `programmatic`

### AC-3: FMEA 基本功能
- **Given**: 用户在 FMEA 模块
- **When**: 添加一条失效模式记录，填写 S/O/D 评分
- **Then**: RPN 自动计算，Action Priority 自动判定
- **Verification**: `programmatic`

### AC-4: Weibull 分析功能保留
- **Given**: 用户在寿命数据分析模块
- **When**: 导入试验数据并选择 Weibull 分布拟合
- **Then**: 显示 β、η、B10 值，绘制概率图和 CDF 图
- **Verification**: `programmatic`

### AC-5: 样本量计算
- **Given**: 用户在测试计划模块
- **When**: 输入目标 B10、置信度、截尾类型
- **Then**: 计算出所需样本量和建议试验时长
- **Verification**: `programmatic`

### AC-6: 故障树构建
- **Given**: 用户在 FTA 模块
- **When**: 添加顶事件并添加下级 AND/OR 门和基本事件
- **Then**: 树形图正确展示，顶事件概率自动计算
- **Verification**: `human-judgment`

### AC-7: 数据跨模块联动
- **Given**: 用户在 FMEA 中定义了失效模式
- **When**: 切换到测试计划模块
- **Then**: 可以选择 FMEA 中的失效模式作为测试项
- **Verification**: `programmatic`

### AC-8: 数据导入导出
- **Given**: 用户有项目数据
- **When**: 点击导出 JSON，然后在新环境中导入
- **Then**: 所有模块数据完整恢复
- **Verification**: `programmatic`

### AC-9: 响应式布局
- **Given**: 用户使用不同屏幕尺寸
- **When**: 调整窗口宽度
- **Then**: 导航栏在小屏自动折叠为侧边抽屉
- **Verification**: `human-judgment`

### AC-10: 数据持久化
- **Given**: 用户完成操作
- **When**: 刷新页面
- **Then**: 所有数据保留，当前选中模块和项目恢复
- **Verification**: `programmatic`
