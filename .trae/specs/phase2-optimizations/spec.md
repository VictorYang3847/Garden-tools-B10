# 可靠性工具平台优化（第二阶段）- Product Requirement Document

## Overview
- **Summary**: 基于《锂电产品可靠性网站设计指导书》，对现有可靠性工具平台进行第二阶段优化，包括新增首页、样本量计算器、短板分析、公式说明等4项核心功能，提升用户体验和专业度。
- **Purpose**: 解决新用户上手难、缺少快速计算入口、深度分析功能不足等问题，让网站更贴近行业最佳实践。
- **Target Users**: 研发工程师、测试工程师、质量工程师、项目经理（锂电/电动工具行业）

## Goals
- 新增首页/快速计算页面，降低新用户门槛
- 新增独立样本量计算器，填补工具空白
- 在寿命分析中增加短板分析（竞争失效）功能
- 为核心计算结果增加公式说明，提升教育价值
- 保持现有10个模块的功能不变，增量式优化

## Non-Goals (Out of Scope)
- 不重构现有架构（保持原生JS + Hash路由）
- 不引入新的第三方图表库（保持Canvas绘图）
- 不做云端同步/多用户功能
- 不做知识库/案例库（后续阶段再考虑）
- 不做导出Excel/Word功能（后续阶段再考虑）

## Background & Context
现有网站已具备10个可靠性模块，但存在以下问题：
1. 没有首页，打开直接进入FMEA，新用户不知所措
2. 缺少独立的样本量计算工具（散落在测试计划中）
3. 寿命分析缺少竞争失效/短板分析功能
4. 计算结果只有数值，没有公式说明
用户提供了详细的《锂电产品可靠性网站设计指导书》，明确了各模块的最佳实践。

## Functional Requirements

### FR-1: 首页/快速计算页面
- 路由：`#/home`（默认路由）
- 包含：欢迎语 + 8个核心工具快捷入口卡片
- 包含：B10快速计算器（直接在首页可用）
- 包含：快速开始引导（3步说明）
- 侧边栏新增"首页"导航项（放在最上面）

### FR-2: 样本量计算器
- 路由：`#/sample-size`
- 侧边栏新增导航项（放在测试工具组）
- 两种计算模式Tab切换：
  - 模式一：合格性验证（二项分布）
  - 模式二：寿命测定（威布尔拟合）
- 实时计算，输入即出结果
- 提供行业参考值对比

### FR-3: 短板分析（竞争失效）
- 在寿命数据分析模块新增"短板分析"Tab
- 按失效模式分组，分别拟合威布尔分布
- 展示各失效模式的B10对比柱状图
- 展示失效模式占比帕累托图/饼图
- 自动识别"制约整机寿命的核心短板"
- 使用默认案例数据验证功能

### FR-4: 公式说明折叠区
- 在寿命分析、可靠性预测、样本量计算等模块的核心结果下方
- 添加"📐 查看公式"折叠按钮
- 展开显示：计算公式、变量说明、代入数值的计算过程
- 使用纯HTML/CSS实现，不依赖额外库
- 至少覆盖：B10计算公式、MTBF计算公式、可靠度计算公式、二项分布样本量公式

## Non-Functional Requirements

### NFR-1: 性能
- 首页加载时间 < 1秒
- 样本量计算实时响应（< 100ms）
- 短板分析计算 < 500ms（6个失效模式以内）

### NFR-2: 兼容性
- 保持纯前端实现，无需后端
- 兼容现有数据结构（不破坏localStorage数据）
- 响应式布局，支持桌面端和Pad

### NFR-3: 可维护性
- 新模块遵循现有代码风格和目录结构
- 计算逻辑与UI分离
- 新增代码有清晰注释

## Constraints
- **Technical**: 原生HTML/CSS/JS + ES Modules，无框架依赖，Canvas绘图
- **Business**: 保持深色主题风格，与现有设计语言一致
- **Dependencies**: 不引入新的第三方库

## Assumptions
- 用户已经熟悉基本可靠性概念（B10、MTBF等）
- 默认案例中的绿篱剪数据可用于验证新功能
- 现有计算引擎（calculator.js）可以复用

## Acceptance Criteria

### AC-1: 首页正常访问
- **Given**: 用户打开网站
- **When**: 访问默认路由或点击侧边栏"首页"
- **Then**: 显示首页，包含欢迎语、8个工具卡片、B10快速计算器
- **Verification**: `programmatic`

### AC-2: B10快速计算器可用
- **Given**: 用户在首页
- **When**: 输入β和η值
- **Then**: 实时显示B10、MTBF、指定时间可靠度等结果
- **Verification**: `programmatic`

### AC-3: 样本量计算器（合格性验证）
- **Given**: 用户在样本量计算器页面，选择"合格性验证"模式
- **When**: 输入可靠度90%、置信度90%、允许失效数0
- **Then**: 计算出最少样本量≈22台，并显示二项分布公式
- **Verification**: `programmatic`

### AC-4: 样本量计算器（寿命测定）
- **Given**: 用户在样本量计算器页面，选择"寿命测定"模式
- **When**: 输入目标精度±20%、置信度90%、预期失效比例60%、β=2.2
- **Then**: 给出推荐样本量和预期失效数
- **Verification**: `human-judgment`

### AC-5: 短板分析Tab存在
- **Given**: 用户进入寿命数据分析模块
- **When**: 查看顶部Tab
- **Then**: 有"短板分析"Tab，点击后显示竞争失效分析内容
- **Verification**: `programmatic`

### AC-6: 按失效模式分别拟合
- **Given**: 寿命数据中有多种失效模式
- **When**: 进入短板分析Tab
- **Then**: 系统自动按失效模式分组，分别拟合威布尔分布
- **Verification**: `programmatic`

### AC-7: 短板可视化
- **Given**: 有多个失效模式的拟合结果
- **When**: 查看短板分析页面
- **Then**: 显示B10对比柱状图和失效模式占比饼图/帕累托图
- **Verification**: `human-judgment`

### AC-8: 公式说明折叠区
- **Given**: 用户在寿命分析的"分析结果"Tab
- **When**: 点击"📐 查看公式"按钮
- **Then**: 展开显示B10、MTBF、可靠度等计算公式和变量说明
- **Verification**: `programmatic`

### AC-9: 默认数据兼容
- **Given**: 用户首次打开网站（有默认案例）
- **When**: 浏览各模块
- **Then**: 所有新功能都有默认数据可以展示，不是空白页
- **Verification**: `programmatic`
