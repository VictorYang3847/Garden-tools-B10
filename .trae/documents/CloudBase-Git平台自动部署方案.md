# CloudBase Git 平台自动部署方案

> 使用 CloudBase 控制台内置的「Git 平台部署」功能，实现 push 代码后自动部署

---

## 一、方案概述

### 当前状态（从截图看到）
- ✅ GitHub 已授权
- ✅ 仓库已选择：`VictorYang3847/Garden-tools-B10`
- ✅ 分支已选择：`main`
- ⬜ 项目名称未填写
- ⬜ 目标目录未配置

### 方案思路

**静态网站**：使用 CloudBase 控制台的「Git 平台部署」功能，push 到 main 分支后自动部署。

**云函数**：使用 GitHub Actions，但不使用 API 密钥，而是通过 CloudBase 提供的其他安全方式。

---

## 二、为什么这个方案更安全？

| 方案 | 安全性 | 复杂度 |
|------|--------|--------|
| API 密钥 + GitHub Secrets | 中（密钥可能泄露） | 中等 |
| CloudBase Git 平台部署 | 高（腾讯云托管，无需暴露密钥） | 低 |

---

## 三、配置步骤

### 步骤 1：配置静态网站 Git 部署（你操作）

在截图的页面上操作：

1. **项目名称**：填一个名字，比如 `reliability-tool-web`
2. **项目框架**：选择「其他」（我们是纯静态 HTML/CSS/JS）
3. **运行时环境**：选择「Node.js」→「Node.js 18」
4. **目标目录**：填 `/`（根目录）
5. **点击「部署」**

> 💡 注意：如果直接部署根目录，会把 `.git`、`cloudbase/` 等所有文件都部署上去。我们需要配置忽略规则。

### 步骤 2：配置忽略规则（我来做）

创建 `.tcbignore` 文件，告诉 CloudBase 不要部署哪些文件：

```
.git/
cloudbase/
worker/
deploy/
node_modules/
.github/
*.md
*.json
```

### 步骤 3：配置云函数自动部署（需要讨论）

CloudBase 的 Git 平台部署功能主要是为静态网站设计的，**不支持自动部署云函数**。

云函数的自动部署有两种选择：

#### 选择 A：手动部署云函数（简单但不自动）
每次修改云函数代码后，手动执行：
```powershell
tcb fn deploy api
```

#### 选择 B：使用 GitHub Actions + 临时登录（需要 API 密钥）
这就是之前的方案，需要配置 API 密钥到 GitHub Secrets。

#### 选择 C：使用云函数在线编辑（不推荐）
在控制台直接编辑云函数代码，不通过 Git 管理。

---

## 四、推荐方案

### 当前阶段推荐：选择 A + 手动部署云函数

理由：
1. 云函数代码不经常变动
2. 静态网站是主要的更新内容
3. 不需要配置任何密钥，最安全

### 未来优化：选择 B（当云函数更新频繁时）

如果以后云函数更新频繁，可以再配置 GitHub Actions。

## 五、具体操作

### 第一步：我创建 .tcbignore 文件

创建 `.tcbignore` 文件，排除不需要部署的文件。

### 第二步：你在控制台配置部署

按步骤 1 的说明配置，然后点击「部署」。

### 第三步：测试自动部署

1. 修改 `index.html` 的标题
2. commit 并 push 到 GitHub
3. 在 CloudBase 控制台查看部署状态
4. 访问网站，确认更新生效

---

## 六、注意事项

1. **部署目录问题**：如果直接部署根目录，可能会包含不必要的文件。`.tcbignore` 文件可以解决这个问题。
2. **部署时间**：Git 平台部署通常需要 1-3 分钟完成。
3. **失败排查**：如果部署失败，在控制台查看部署日志。
