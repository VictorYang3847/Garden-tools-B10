# 首页 B10/MTBF 快速计算器改进方案

## 摘要

当前首页计算器要求用户输入"特征寿命 η"，但 η 是寿命数据分析后才能拟合出的结果，项目初期不可能知道。本方案将计算器重构为**以业务目标驱动**的逆向计算工具——用户输入质保期、允许失效率等业务参数，反推目标 B10 寿命和所需 MTBF，符合可靠性工程的实际工作流程。

## 现状分析

### 当前实现（home.js + home-template）

**输入项**：
| 字段 | 默认值 | 问题 |
|---|---|---|
| 形状参数 β | 2.2 | 项目初期可凭经验假设，合理 |
| 特征寿命 η (小时) | 180 | **项目初期不可能知道**，需寿命数据拟合 |
| 目标可靠度 (滑块) | 90 | 计算了 `bp` 变量但未显示，属于死输入 |
| 指定时间 t | 40 | 仅用于计算 R(t)，合理但作用有限 |

**输出项**：B10、MTBF、R(t)、F(t)

**核心问题**：
1. **输入逻辑反了**：η 是"结果"不是"输入"。正确流程是：先有业务目标（质保期多长、允许多少失效），再反推需要多大的 B10，最后才知道 η 应该是多少
2. **目标可靠度是死输入**：`bp` 变量被计算但从未显示
3. **与 calculator.js 脱节**：首页自带 gammaApprox，不复用 calculator.js 的 `targetB10()`、`weibullEta()` 等函数，且默认值不一致（首页 β=2.2，calculator 默认 β=2.0）

### 行业实际工作流程

可靠性工程的项目阶段与可用参数：

| 阶段 | 已知输入 | 计算输出 |
|---|---|---|
| **项目初期（定义阶段）** | 质保期、允许失效率、β（经验假设）、安全余量 | 目标 B10、所需 η、所需 MTBF |
| **设计阶段** | 目标 B10、零件 B10 | 系统合成 B10、瓶颈分析 |
| **测试阶段** | 目标 B10、β、置信度 | 样本量、测试时间 |
| **量产阶段（分析阶段）** | 失效数据 | 拟合 β、η、B10（此时才知道真实 η）|

→ 首页计算器应服务于**项目初期**，做"目标定义"计算。

## 改进方案

### 设计思路

将计算器从"已知 η → 算 B10"改为"已知业务目标 → 算所需 B10 和 η"：

**正向（现状）**：η, β → B10, MTBF, R(t)  
**逆向（改进后）**：质保期, 允许失效率, β → 目标 B10 → 所需 η, 所需 MTBF

### 新输入项

| 字段 | id | 默认值 | 说明 |
|---|---|---|---|
| 质保期（年） | `home-warranty-years` | 2 | 产品质保年限 |
| 年使用时长（小时/年） | `home-hours-per-year` | 25 | 用户年均使用小时数 |
| 允许失效率（%） | `home-allow-fail-rate` | 2 | 质保期内允许失效的比例 |
| 形状参数 β | `home-beta` | 2.2 | Weibull 形状参数（经验假设，磨损失效典型 2.0~2.5） |
| 安全余量（%） | `home-safety-margin` | 20 | 设计余量，行业常用 20%~30% |
| 指定时间 t（小时） | `home-time` | 50 | 查询任意时刻的可靠度（可选） |

### 新输出项

| 输出 | 计算公式 | 说明 |
|---|---|---|
| 质保总时长 Tw | `年数 × 年时长` | 中间量 |
| 目标 B10（含余量） | `Tw × (K10 / -ln(1-Fw))^(1/β) × (1+margin)` | 调用 calculator.js 的 `targetB10()` |
| 所需特征寿命 η | `B10 / K10^(1/β)` | 调用 calculator.js 的 `weibullEta()` |
| 所需 MTBF | `η × Γ(1+1/β)` | 仍用 gammaApprox |
| t 时刻可靠度 | `exp(-(t/η)^β)` | 用算出的 η |
| t 时刻失效概率 | `1 - R(t)` | |

### 复用 calculator.js

首页改为导入 calculator.js 中的函数：
- `targetB10(tw, fw, beta, margin)` — 计算含余量的目标 B10
- `targetB10WithoutMargin(tw, fw, beta)` — 不含余量的目标 B10
- `weibullEta(b10, beta)` — 由 B10 反算 η
- `failureRate(t, b10, beta)` — 任意时刻失效概率

首页不再自带 `gammaApprox`，改为从 life-data.js 或 calculator.js 中共享（或将 gammaApprox 提取到 calculator.js 导出）。

### 公式说明区更新

更新为"目标定义"相关公式：
- 质保总时长：`Tw = 质保年数 × 年使用时长`
- 目标 B10：`B10 = Tw × [ln(10/9) / -ln(1-Fw)]^(1/β) × (1+margin)`
- 特征寿命：`η = B10 / [ln(10/9)]^(1/β)`
- MTBF：`MTBF = η × Γ(1+1/β)`
- 时刻可靠度：`R(t) = exp(-(t/η)^β)`

### 1. `js/pages/home.js`
- 移除自带的 `gammaApprox`（改为导入）
- 导入 calculator.js 的 `targetB10`、`targetB10WithoutMargin`、`weibullEta`、`failureRate`

- 重写 `calculate()`：实现逆向计算逻辑
- 保留 `gammaApprox`（从 calculator.js 导入或内部保留一份）

### 2. `index.html`（home-template 部分，约第 168-265 行）
- 修改 `.b10-calc-inputs` 区域：移除 η 输入，添加质保期、年使用时长、允许失效率、安全余量输入
- 修改 `.b10-calc-results` 区域：输出项改为目标 B10（高亮）、所需 η、所需 MTBF、t 时刻可靠度、t 时刻失效概率

### 3. `js/calculator.js`
- 导出 `gammaApprox` 函数（供 home.js 复用），或新增一个 `calcMtbf(eta, beta)` 工具函数

## 假设与决策

- **β 默认值**：保持 2.2（与现有首页一致，且在 calculator.js 的可靠性分配子模块中也使用 2.2）
- **gammaApprox 复用方式**：将 gammaApprox 提取到 calculator.js 中导出，避免代码重复
- **输入校验**：质保期 0.5~10 年，年使用时长 1~1000 小时，允许失效率 0.1%~10%，安全余量 0%~50%
- **不引入"目标可靠度"滑块**：当前"允许失效率"已经表达了同样信息，避免冗余
